\subsection{Introduction}

Information technology has reshaped much of university life. Admission, registration, exam scheduling, and result publication have all moved online at most institutions. Yet one part of the process has stubbornly resisted digitization: the evaluation of handwritten, descriptive admission answer scripts. At many universities, this step still runs the old way -- teachers sort through stacks of paper, mark them by hand, cross-check with moderators, and manually compile the results. The process eats up time, demands heavy manual labor, and is hard for administrators to track or supervise \cite{article,inproceedings}.

The scale of the problem becomes obvious during large admission tests. Examiners from different departments -- Physics, Chemistry, Mathematics, English -- all need to grade scripts within a tight window. Because everything is done physically, from handing out scripts to logging marks by hand, the workflow is prone to delays and simple human error \cite{8878791}. On top of that, keeping grading standards consistent across many examiners gets harder as the number of applicants climbs, and there's no easy way to see how far along the evaluation actually is at any given moment.

Fortunately, developments in web technology, document image processing, and Optical Character Recognition (OCR) open a path forward. Tools like Tesseract can already pull printed text out of scanned pages, and newer document-analysis methods make it possible to handle large volumes of documents automatically \cite{4376991,FRANCIS2025100532}. These tools have made a real dent in document digitization generally, but they can't replace the human judgment that grading a descriptive, handwritten answer still requires.

This is the gap that motivates the current work. This thesis introduces the \textbf{Digital Admission Script Evaluation and Moderation System (DASEMS)}, a web platform built to digitize the admission-script evaluation process from start to finish. It handles secure PDF uploads, splits scripts into questions using a predefined template, keeps students anonymous through student identification codes (\textbf{s\_code}), routes each question to the right subject teacher automatically, lets teachers grade directly in the browser, supports moderation by Head Examiners, and produces a centralized result at the end. The goal is not to remove teachers from the loop, but to make the whole process faster, more transparent, and fairer while leaving the actual academic judgment where it belongs -- with the evaluators.

\subsection{Background and Motivation}

Admission examinations sit near the top of the list of high-stakes assessments in higher education. Thousands of candidates sit these exams every year, and teachers are left to grade a large volume of descriptive answers in a short amount of time. Even at universities that have digitized much of their exam administration, the actual marking of handwritten descriptive answers is often still done on paper \cite{article}.

The typical process looks like this: scripts get physically handed out to teachers, marked by hand, checked over by senior examiners, and finally added up into result sheets. Managing this by hand takes considerable administrative effort, and it's genuinely difficult to track progress, guarantee anonymity, or keep grading consistent across multiple examiners \cite{8878791}. Paper also brings its own risks -- scripts get misplaced, totals get miscalculated, and results end up delayed.

Work in document image processing, OCR, and layout analysis has already shown that a lot of document handling can be automated effectively \cite{4376991,inbook,docbank2020}. These techniques make it feasible to process scanned documents electronically and lay the groundwork for smarter exam-management systems. That said, most existing systems either focus narrowly on digitizing documents or on running online exams -- not on supporting the full pipeline of descriptive admission-script evaluation across multiple teachers and a moderation stage.

What this research sets out to do is build something practical: a web platform that cuts down administrative overhead without taking human evaluation out of the equation. The system takes in scanned answer scripts, breaks them into question-wise regions using a set template, hands questions off automatically to the relevant subject teachers, allows anonymous browser-based grading, and steps in with a moderation mechanism whenever examiners disagree significantly.

\subsection{Research Objectives}

The core aim of this research is to build a secure, efficient web platform for evaluating descriptive university admission scripts. It targets automation of document processing, teacher assignment, evaluation management, moderation, and result generation -- all in service of a process that is faster, more transparent, and fairer than the current manual one.

\subsubsection{General Objective}

The general objective is to design and build a \textbf{Digital Admission Script Evaluation and Moderation System (DASEMS)} that manages the entire descriptive admission-script evaluation workflow, end to end.

\subsubsection{Specific Objectives}

\begin{itemize}

\item Build a secure, role-based platform for Super Administrator, Head Examiner, and Teacher roles.

\item Introduce anonymous student identification via a unique \textbf{s\_code}, so evaluation stays unbiased.

\item Process uploaded admission scripts and split them into question-wise regions automatically, using template-based document segmentation.

\item Build an automated mechanism that routes answers to the appropriate subject teachers.

\item Provide a browser-based evaluation interface with support for digital annotation, comments, and mark entry.

\item Generate centralized evaluation reports and final results once moderation is complete.

\end{itemize}

\subsection{Scope of the Research}

This research is centered on building a web platform that digitizes the evaluation of handwritten university admission answer scripts. It covers the full workflow -- student information management, script upload, question-wise segmentation, automatic teacher assignment, digital evaluation, moderation, and result generation.

The scope includes:

\begin{itemize}

\item A role-based web application for Super Administrator, Head Examiner, and Teacher.

\item Processing scanned admission scripts in PDF format via template-based document segmentation.

\item Automatically splitting uploaded scripts into individual question-wise answer regions.

\item Anonymous evaluation through a unique student identification code (\textbf{s\_code}).

\item Automatic assignment of question-wise evaluation tasks to subject-specific teachers.

\item A browser-based evaluation interface supporting digital annotation, comments, and mark submission via a graphics tablet or stylus.

\item A three-examiner evaluation workflow with moderation support for conflicting marks.

\item Final result generation once evaluation and moderation are complete.

\end{itemize}

This work does not try to replace human judgment in grading descriptive answers. The focus stays on making the evaluation workflow more efficient and better managed, while leaving room for AI-assisted assessment to be layered in later.

\subsection{Research Contributions}

\begin{itemize}

\item A web-based Digital Admission Script Evaluation and Moderation System (DASEMS) that supports the complete handwritten admission-script evaluation workflow.

\item A template-based document segmentation approach that automatically splits uploaded scripts into question-wise regions while keeping the original scanned document intact.

\item An anonymous evaluation mechanism built around a unique student identification code (\textbf{s\_code}), aimed at keeping assessment fair and unbiased.

\item An automated mechanism for distributing evaluation tasks among subject-specific teachers.

\item A browser-based digital evaluation interface with graphical annotation, comments, and mark submission, built to mirror traditional paper-based marking.

\item A three-examiner moderation framework that flags significant marking differences and supports the Head Examiner's final decision.

\item A centralized result-generation mechanism that ties evaluated marks to student information and produces the final examination results.

\end{itemize}
